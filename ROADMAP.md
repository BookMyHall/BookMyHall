# BookMyHall Roadmap

This file outlines the high-level roadmap and milestones.

## Phase 1 — Static MVP (Complete)
- Static Client, Vendor and Admin sites for demos
- Basic booking flow (static mock)
- Marketing landing page

## Phase 2 — Core Platform (Next 3 months)
- FastAPI backend: authentication, CRUD for venues, bookings
- Next.js client with dynamic booking flows
- Vendor dashboard with availability management
- PostgreSQL database and Dockerized services

## Phase 3 — Payments & Mobile (3–6 months)
- Stripe integration for payments and payouts
- Mobile app prototype (React Native / Expo)
- Role-based access control and admin reports

## Phase 4 — Scale & Polish (6–12 months)
- Real-time updates (websockets) for booking status
- Monitoring, logging & SRE improvements
- Marketplace analytics and recommendations

## Open items & stretch goals
- Multi-currency and localization
- Calendar sync (Google, Outlook)
- Automated contract generation & e-signature
